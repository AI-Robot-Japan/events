# Contributing to AI & Robotics Events Japan

日本語版は下にあります / Japanese version below

## English

Thank you for your interest in contributing to this project! We welcome contributions from everyone.

### How to Contribute

1. **Fork the repository**
   - Click the "Fork" button at the top right of this page

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/events.git
   cd events
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes**
   - Follow the coding guidelines in `.clinerules`
   - Write clear commit messages
   - Add tests if applicable

5. **Test your changes**
   - Ensure all tests pass
   - Test manually if needed

6. **Commit your changes**
   ```bash
   git add .
   git commit -m "Add: Brief description of your changes"
   ```

7. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Create a Pull Request**
   - Go to the original repository
   - Click "New Pull Request"
   - Select your branch
   - Provide a clear description of your changes

### Commit Message Guidelines

Use the following prefixes:
- `Add:` - New features or files
- `Fix:` - Bug fixes
- `Update:` - Updates to existing features
- `Remove:` - Removal of features or files
- `Docs:` - Documentation changes
- `Style:` - Code style changes (formatting, etc.)
- `Refactor:` - Code refactoring
- `Test:` - Adding or updating tests

### Code Review Process

1. Maintainers will review your PR
2. Address any feedback or requested changes
3. Once approved, your PR will be merged

## 日本語

このプロジェクトへの貢献に興味を持っていただき、ありがとうございます！皆様からの貢献を歓迎します。

### 貢献方法

1. **リポジトリをフォーク**
   - このページの右上にある「Fork」ボタンをクリック

2. **フォークをクローン**
   ```bash
   git clone https://github.com/YOUR_USERNAME/events.git
   cd events
   ```

3. **ブランチを作成**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **変更を加える**
   - `.clinerules`のコーディングガイドラインに従う
   - 明確なコミットメッセージを書く
   - 該当する場合はテストを追加

5. **変更をテスト**
   - すべてのテストが通ることを確認
   - 必要に応じて手動でテスト

6. **変更をコミット**
   ```bash
   git add .
   git commit -m "Add: 変更の簡単な説明"
   ```

7. **フォークにプッシュ**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **プルリクエストを作成**
   - 元のリポジトリに移動
   - 「New Pull Request」をクリック
   - ブランチを選択
   - 変更内容の明確な説明を提供

### コミットメッセージのガイドライン

以下のプレフィックスを使用してください：
- `Add:` - 新機能やファイルの追加
- `Fix:` - バグ修正
- `Update:` - 既存機能の更新
- `Remove:` - 機能やファイルの削除
- `Docs:` - ドキュメントの変更
- `Style:` - コードスタイルの変更（フォーマットなど）
- `Refactor:` - コードのリファクタリング
- `Test:` - テストの追加または更新

### コードレビュープロセス

1. メンテナーがPRをレビューします
2. フィードバックや変更要求に対応してください
3. 承認されると、PRがマージされます

## Questions? / 質問がありますか？

If you have any questions, please open an issue or contact the maintainers.
質問がある場合は、issueを開くか、メンテナーに連絡してください。

Thank you for contributing! / 貢献ありがとうございます！

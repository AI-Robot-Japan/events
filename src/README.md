# Source Code / ソースコード

This directory contains the source code for the project.
このディレクトリには、プロジェクトのソースコードが含まれています。

## Structure / 構造

```
src/
├── README.md        # This file / このファイル
└── (source files)   # Source files will be added here / ソースファイルはここに追加されます
```

## Getting Started / はじめに

Source code will be organized based on the project requirements.
ソースコードは、プロジェクト要件に基づいて整理されます。

Example organization:
組織の例：

```
src/
├── models/         # Data models / データモデル
├── services/       # Business logic / ビジネスロジック
├── utils/          # Utility functions / ユーティリティ関数
├── api/            # API endpoints / APIエンドポイント
└── config/         # Configuration / 設定
```

## Guidelines / ガイドライン

When adding source code:
ソースコードを追加する際：

1. Follow the coding guidelines in `.clinerules` / `.clinerules`のコーディングガイドラインに従う
2. Add appropriate tests in the `tests/` directory / `tests/`ディレクトリに適切なテストを追加
3. Document your code with comments / コメントでコードをドキュメント化
4. Update relevant documentation / 関連ドキュメントを更新

## Code Style / コードスタイル

- Use consistent naming conventions / 一貫した命名規則を使用
- Keep functions small and focused / 関数は小さく、集中させる
- Write self-documenting code / 自己ドキュメント化コードを書く
- Add comments for complex logic / 複雑なロジックにはコメントを追加
